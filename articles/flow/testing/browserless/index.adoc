= Browserless Testing

[NOTE]
Browserless testing was previously known as _UI Unit Testing_ and required a commercial TestBench subscription. Starting with Vaadin 25.1, browserless testing is free for all users.

Browserless testing removes the need to run a browser or a servlet container, making it much faster to test your Vaadin-based applications.

With browserless testing, you look up components directly from [methodname]`UI.getCurrent()`, bypassing the browser and the client-server communication bridge. You call methods directly on your server-side view classes and Vaadin components.

You also don't need to launch a servlet container. The [classname]`BrowserlessTest` class creates the Vaadin session, the UI, and other necessary Vaadin classes directly in the JVM that runs your JUnit tests.

In addition to browserless testing, Vaadin also allows you to write end-to-end tests for your applications. <<../#comparison,Each approach has its own advantages>>.
