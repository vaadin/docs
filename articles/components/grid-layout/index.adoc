= Grid Layout

// tag::description[]
Grid Layout arranges components in a two-dimensional grid using CSS Grid.
// end::description[]
It provides a Java API for configuring grid columns, rows, gaps, and item placement. Unlike <<../vertical-layout#,Vertical Layout>> and <<../horizontal-layout#,Horizontal Layout>> which use flexbox, Grid Layout uses CSS Grid, making it ideal for complex two-dimensional layouts.

.Flow-only component
[NOTE]
====
Grid Layout is currently available only for Flow (Java). For Lit, React, or plain HTML, use a `<div>` element with CSS `display: grid` directly.
====

== Defining Columns and Rows

Grid Layout allows you to define the structure of your grid using template columns and rows.


=== Column Templates

Use `setColumns()` to define the column structure. You can specify column widths using CSS Grid syntax:

[.example]
--
ifdef::flow[]
[source,java]
----
include::{root}/src/main/java/com/vaadin/demo/component/gridlayout/GridLayoutColumns.java[render,tags=snippet,indent=0,group=Flow]
----
endif::[]
--

Common column patterns:

|===
|Pattern |Description

|`"1fr 1fr 1fr"`
|Three equal-width columns using fractional units.

|`"200px auto 200px"`
|Fixed sidebars with flexible center.

|`"repeat(3, 1fr)"`
|Shorthand for three equal columns.

|`"repeat(auto-fill, minmax(200px, 1fr))"`
|Responsive columns that wrap automatically.

|`setColumns(3)`
|Convenience method for three equal columns.
|===

== Alignment

Grid Layout provides several alignment options for controlling how items are positioned within their grid cells.


=== Item Alignment

Control how all items are aligned within their cells:

[.example]
--
ifdef::flow[]
[source,java]
----
include::{root}/src/main/java/com/vaadin/demo/component/gridlayout/GridLayoutAlignment.java[render,tags=snippet,indent=0,group=Flow]
----
endif::[]
--

|===
|Property |Values |Description

|`setJustifyItems()`
|`START`, `END`, `CENTER`, `STRETCH`, `BASELINE`
|Horizontal alignment of items within cells.

|`setAlignItems()`
|`START`, `END`, `CENTER`, `STRETCH`, `BASELINE`
|Vertical alignment of items within cells.
|===


=== Individual Item Alignment

=== Content Alignment

When the grid is smaller than its container, control how the entire grid is positioned:

|===
|Property |Values |Description

|`setJustifyContent()`
|`NORMAL`, `START`, `END`, `CENTER`, `STRETCH`, `SPACE_BETWEEN`, `SPACE_AROUND`, `SPACE_EVENLY`
|Horizontal alignment of the grid within the container.

|`setAlignContent()`
|`NORMAL`, `START`, `END`, `CENTER`, `STRETCH`, `SPACE_BETWEEN`, `SPACE_AROUND`, `SPACE_EVENLY`
|Vertical alignment of the grid within the container.
|===



== Auto Flow

Control how auto-placed items flow into the grid:

[.example]
--
ifdef::flow[]
[source,java]
----
include::{root}/src/main/java/com/vaadin/demo/component/gridlayout/GridLayoutAutoFlow.java[render,tags=snippet,indent=0,group=Flow]
----
endif::[]
--

|===
|Value |Description

|`ROW` (default)
|Items fill each row before moving to the next.

|`COLUMN`
|Items fill each column before moving to the next.

|`ROW_DENSE`
|Items fill rows, attempting to fill gaps earlier in the grid.

|`COLUMN_DENSE`
|Items fill columns, attempting to fill gaps earlier in the grid.
|===

== Best Practices

=== When to Use Grid Layout

Grid Layout is best suited for:

* Two-dimensional layouts where both rows and columns matter
* Page layouts with header, sidebar, main content, and footer areas
* Dashboard-style layouts with cards of varying sizes
* Any layout where items need to span multiple rows or columns

For simpler one-dimensional layouts, consider using <<../vertical-layout#,Vertical Layout>> or <<../horizontal-layout#,Horizontal Layout>> instead.


=== CSS Grid vs. Flexbox

|===
|Use Grid Layout when... |Use Vertical/Horizontal Layout when...

|You need control over both rows and columns
|Content flows in one direction

|Items need to span multiple rows/columns
|Items simply stack or line up

|You're creating a page-level layout
|You're laying out a component's internal structure

|You need named template areas
|You need simple alignment and spacing
|===
