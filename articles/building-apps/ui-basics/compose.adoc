= Compose with Components
:toclevels: 2

* Introduction to this article. It's purpose is to teach thinking in components, not show how to implement them in code.

== What Is a Component?

* Vaadin UIs are built from components.
* Components are like pieces of legos, by combining them you can build various kinds of UIs.
* Components have a server side Java component and a client-side HTML element
* Technical details about how components work: <</flow/component-internals#,Component Internals Reference>>
* The most primitive components are basic HTML elements like `<div>`, `<span>`, `<h1>`, and `<p>`.
* Vaadin provides built-in rich components for common use cases: <</components#,Components documentation>>
* You often end up creating your own components as well.
  * Components can be one-off context specific or reusable

== Views Are Components

* A view is a user interface component that forms a logical whole of the user interface. A page can contain other components than a view, but only one view at a time. Typically, the view takes up most of the page. See the <<../views/add-view#,Add a View>> guide for more info
* Views are not meant to be reused or extended.
* For simple or flat views, build the entire view directly in the view's constructor.
* For more complex views, you should split the view into smaller components

== State and Actions

* The view is responsible for maintaining state and calling application <<../business-logic/add-service#,services>>.
* The components of the view should be fairly flat and agnostic of the business logic.
* The view updates the state of the components through setter methods or <</flow/ui-state#,signals>>
* The components inform the view of changes through <</flow/component-internals/events#,events>> or callbacks
